# 縦切り実装手順 (Vertical Slice)

機能は水平 (全スキーマ → 全サーバー → 全UI) ではなく、1機能ずつ縦に貫通させて完成させる。

## 手順

1. **計画**: 影響ファイルを列挙し、ユーザーに 3 行以内で計画を提示する
2. **shared**: op の型・zod スキーマ・reducer (`applyOp`) を `shared/` に追加
3. **テスト先行**: reducer とスキーマのテストを先に書き、失敗を確認する
4. **DO**: `worker/board-do.ts` で受信 → 検証 → 適用 → 永続化 → ブロードキャストを実装
5. **クライアント**: 楽観的適用 + 送信 + 受信反映、UI/演出を DESIGN.md 準拠で実装
6. **UI の自己検証**: UI を実装したら playwright MCP (.mcp.json 設定済み) のブラウザで
   実際に開き、スクリーンショットを撮って design/DESIGN.md との一致
   (質感・レイアウト・レスポンシブ) を自分の目で確認する。確認は
   375px / 768px / 1920px の 3 幅で行う
7. **検証**: `pnpm typecheck && pnpm lint && pnpm test` を実行し、出力を証拠として報告。
   同期が絡む機能は 2 タブでの動作確認も行い内容を報告する
8. **レビュー**: code-reviewer エージェントに差分レビューを依頼し、[must] 指摘をすべて解消

## 禁止事項

- 手順 3 を飛ばしてテストを後付けにすること
- 複数機能の同時着手 (1 スライス完了まで次に進まない)
- 検証結果を貼らずに「完了しました」と報告すること
